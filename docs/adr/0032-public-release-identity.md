# 32. Public release identity: name, independence, and mark

Status: Accepted
Date: 2026-07-11

## Context

The project ships to the Chrome Web Store and Firefox Add-ons for the first time in
the release readiness phase. Through development the product carried the working name
KMITL Course Planner, and the icon was the placeholder generated by the project
scaffold. Both stores enforce impersonation and intellectual property rules: a listing
must not imply an affiliation with, or an endorsement by, a party it is not connected
to, and it must not reuse that party's marks. The tool reads the public course data of
one university and is built by a student with no connection to that university's
registrar or administration.

A name that leads with the institution reads as an official product of the institution.
A name that leads with the function and names the institution as the context it serves
reads as an independent tool for use at that institution, which is what this is. The
same reasoning applies to the mark: the university seal or logo would imply an official
origin, so the mark must be original.

## Decision

The extension presents as Course Planner for KMITL, with the short form KCP. The manifest
name, the launcher label copy, and the in interface English title all use this name. The
Thai interface keeps its descriptive title, since it already reads as a schedule planner
for the institution rather than an official product of it.

The listing, the README, and the privacy statement each carry a plain independence
notice: this is an independent student tool, not affiliated with, sponsored by, or
endorsed by King Mongkut's Institute of Technology Ladkrabang.

The icon is an original mark, a white weekly grid with one filled slot on the project
orange, and is never the university seal or logo. The mark is kept as `assets/icon.svg`
and rendered to the shipped `public/icon` sizes.

The internal package name, the repository name, and the add-on gecko id keep their
existing `kmitl-course-planner` form, because they are identifiers rather than a
user facing display name and changing them would break the signing identity and the
release tooling for no gain.

## Alternatives considered

Keeping the name KMITL Course Planner was rejected because leading with the institution
name raises an avoidable impersonation review risk on both stores and misstates the
relationship. Renaming the package and repository as well was rejected because those are
stable identifiers wired into the gecko id and the release artifact names, and moving
them buys nothing the display name change does not already buy.

## Consequences

The stores see a functional, clearly independent name with a matching disclaimer and an
original mark, which is the intended posture for review. A reader who knows only the
former name can still find the project through the unchanged repository and package
identifiers. If a store reviewer still objects to the institution appearing in the name,
the fallback is to drop to the bare KCP form, recorded here so the decision path is
already mapped.
